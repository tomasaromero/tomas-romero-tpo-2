Optimización de rendimiento en SPAs

22 de Octubre, 2025

Las Aplicaciones de Una Sola Página (SPAs), construidas con frameworks como React, Vue o Angular, ofrecen una experiencia de usuario (UX) increíblemente fluida. Sin embargo, esta fluidez tiene un costo inicial: un archivo JavaScript (el bundle) grande que debe ser descargado antes de que el usuario vea algo. Este es el principal cuello de botella.

1. Code Splitting (División de Código)

Por defecto, los bundlers empaquetan toda tu aplicación en un solo archivo gigante. El Code Splitting es la técnica de dividir ese bundle en trozos más pequeños.

División por Rutas (Lazy Loading)

Es la forma más efectiva. Cargamos el código de una ruta solo cuando el usuario navega hacia ella.

Ejemplo en React:

// En lugar de: import Perfil from './paginas/Perfil';
// Usamos Lazy Loading:

import React, { Suspense, lazy } from 'react';

// El componente se carga solo cuando se necesita
const Perfil = lazy(() => import('./paginas/Perfil'));

function App() {
  return (
    // Mostramos un spinner mientras se carga el chunk
    <Suspense fallback={<div>Cargando...</div>}>
      <Perfil />
    </Suspense>
  );
}

2. Tree Shaking (Optimización de Árbol)

El Tree Shaking es un proceso que elimina el "código muerto" (código que importamos pero que nunca usamos) de nuestro bundle final.

3. Caching Estratégico y Service Workers

Una vez que el usuario ha descargado un archivo, ¡no deberíamos obligarlo a descargarlo de nuevo!

  • Hashing de archivos: Los archivos se nombran con un hash (ej. main.a8b3c9f.js). Si el código no cambia, el navegador usa la versión en caché.
  • Service Workers: Permiten un control total del caché, habilitando experiencias offline y cargas instantáneas en visitas recurrentes.

4. Optimización de Assets (Imágenes y Fuentes)

A menudo, el JavaScript no es el único culpable. Imágenes pesadas pueden bloquear el renderizado.

Usa formatos modernos (WebP, AVIF) que ofrecen mejor compresión. Implementa la carga diferida (lazy loading) para imágenes que están "fuera de pantalla".

Conclusión

El rendimiento de las SPAs es un equilibrio. Aplicando Code Splitting y optimizando assets, podemos obtener la fluidez de una SPA sin la penalización de una carga inicial lenta.
